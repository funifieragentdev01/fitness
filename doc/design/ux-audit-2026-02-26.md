## 1. Resumo Executivo

O app tem uma base sólida de funcionalidades (onboarding, planos IA, gamificação), mas está funcionando como **MVP técnico**, não como **produto comercial**. As maiores lacunas estão na **conversão** (falta paywall/preços), na **retenção** (fluxo do desafio 90 dias quebrado), e na **experiência visual** (telas internas ainda com cara de protótipo).

## 3. Problemas Críticos de UX (por tela)

### 🔴 3.1 Landing / Home Page
**Estado atual:** Nova hero com bg.jpeg, seções de features, depoimentos.

**Problemas:**
1. **Sem seção de preços** — O doc de business model define R$39,90/mês e R$249,90/ano. A landing precisa de cards de preço com CTA de trial. Sem isso, não existe funil de conversão.
2. **Depoimentos são fictícios** — "Marcos, 32 anos" não tem foto, não tem antes/depois. O design conceito do Freepik mostrava fotos reais de antes/depois. Isso é 10x mais poderoso.
3. **Falta quiz/teaser** — O gamification-design sugere um "quiz de atratividade" na landing: "Descubra o que a IA recomendaria para seu corpo". Isso aumenta conversão significativamente.
4. **Falta "como funciona"** — As 3 etapas (conte sobre você → IA gera planos → evolua) estavam na versão anterior e sumiram.

**Recomendação:** Landing precisa das seções: Hero → Como Funciona (3 steps) → Features → Desafio 90 Dias → Depoimentos (com fotos antes/depois) → Planos/Preços → CTA final.

### 🔴 3.2 Cadastro (signup)
**Estado atual:** Nome, email, usuário, senha.

**Problemas:**
1. **Campo "Usuário" é confuso** — Ninguém espera criar um username separado do email. O padrão do mercado é email = login. Isso adiciona fricção no cadastro.
2. **Falta opção de login social** — Google login é mencionado no UX spec (T02) mas não implementado.
3. **Sem coleta de cartão para trial** — O business model recomenda "coletar cartão no cadastro, cobrar após 7 dias" para 2-3x mais conversão.

**Recomendação:** Remover campo "Usuário", usar email como _id do player. Simplificar para: Nome + Email + Senha + Termos.

### 🔴 3.3 Desafio 90 Dias — Aceitar
**Estado atual:** Na dashboard, clicar em "Aceitar" abre `showChallengeOffer = true` mas o modal/fluxo está quebrado — não redireciona.

**Problemas:**
1. **Botão não funciona** — Clique não faz nada visível.
2. **Faltam os passos do aceite** — Aceitar → primeiro check-in corporal → definir meta → confirmar compromisso.
3. **Card do desafio na dashboard** — Não tem a mesma largura dos outros boxes (XP card, próximas ações). Está dentro de `.section` com padding, enquanto o XP card usa margin.

**Recomendação:** 
- Corrigir o fluxo: Aceitar → redirecionar para `/challenge90` com estado "novo" → CTA "Fazer primeiro check-in" → redirecionar para body-checkin.
- Padronizar largura dos cards na dashboard.

### 🟡 3.4 Check-in Corporal
**Estado atual:** Peso + fotos (frente/lateral) com FileReader → base64 local. "Salvar Check-in" loga action mas fotos não são persistidas (campo `photo_front: 'uploaded'` é apenas string).

**Problemas:**
1. **Fotos NÃO são salvas** — O código salva apenas a string "uploaded", não a URL real. Precisa usar `/v3/upload/image` do Funifier.
2. **Histórico mostra só peso/data** — Falta thumbnail das fotos.
3. **Fluxo confuso** — "Salvar Check-in" depois "Salvar medidas" são dois passos quando deveria ser um.
4. **Sem comparativo antes/depois** — O UX spec descreve slider de comparação (T20) mas não existe.

**Recomendação:** Implementar upload real via Funifier API, salvar URLs no checkin, mostrar fotos no histórico e no desafio 90 dias.

### 🟡 3.5 Evolução (Progress)
**Estado atual:** Mostra TODOS os níveis do sistema em timeline horizontal, XP por semana (bar chart), stats de desafios, histórico de peso.

**Problemas:**
1. **Timeline de níveis mostra tudo** — Confuso e desmotivante ver 10 níveis quando você está no 2. O padrão é mostrar apenas: nível atual + quanto falta pro próximo.
2. **Sem timeline de fotos corporais** — É o recurso visual mais poderoso de retenção (ver a própria evolução).
3. **Sem gráfico de peso** — Só lista texto. Deveria ter Chart.js line chart.
4. **Falta badges** — A tela T23 (Conquistas) do UX spec não existe.

**Recomendação:** Redesenhar como: Card de nível (atual → próximo) → Gráfico de peso → Timeline de fotos → Badges grid → Stats.

### 🟡 3.6 Dashboard
**Estado atual:** Funcional, boa estrutura.

**Problemas:**
1. **Card do desafio 90 dias tem largura diferente** dos outros cards.
2. **Falta streak visual proeminente** — O 🔥 streak é apenas um textinho no footer do XP card. Deveria ser mais visível (como Duolingo).
3. **"Desafios Ativos"** mostra card genérico + card 90 dias duplicado (há dois cards para 90 dias: um no topo e outro em "Desafios Ativos").

## 4. Problemas de Marketing / Conversão

### 4.1 Funil inexistente
```
Landing → Cadastro → Onboarding → App (grátis, sem limite)
```
**Deveria ser:**
```
Landing → Ver Preços → Cadastro (+ cartão) → Trial 7 dias → Onboarding → App → Paywall dia 8
```

### 4.2 Sem monetização
- Não existe tela de preços
- Não existe integração com Stripe/pagamento
- Não existe diferença entre free e premium
- **Impacto:** O app não gera receita. Cada usuário é custo puro (Funifier + OpenAI).

### 4.3 Social proof fraco
- Depoimentos genéricos sem foto
- Sem antes/depois
- Sem número de usuários
- **Solução:** O desafio 90 dias deveria gerar social proof real: foto antes + foto depois + depoimento + nota = conteúdo para landing page.

## 6. O Desafio 90 Dias como Motor de Marketing

Este é o insight mais importante: **o desafio 90 dias não é só uma feature de gamificação — é a principal ferramenta de marketing do produto.**

### Fluxo ideal:
```
Dia 0:  Aceitar desafio → Primeiro check-in (peso + fotos frente/lateral)
        Meta pessoal ("Quero perder 8kg" / "Quero definir abdômen")
        
Dia 30: Check-in intermediário (peso + fotos)
        IA compara com dia 0, mostra evolução
        
Dia 60: Check-in intermediário (peso + fotos)
        IA mostra tendência, projeta resultado final
        
Dia 90: Check-in final (peso + fotos)
        IA faz análise completa da transformação
        Comparativo lado a lado (dia 0 vs dia 90)
        → DEPOIMENTO: texto escrito pelo usuário
        → NOTA: 1 a 5 estrelas
        → PUBLICAR: opt-in para aparecer na landing page
```

### Por que isso é poderoso:
1. **Social proof gerado pelo produto** — não precisa inventar depoimentos
2. **Fotos reais de antes/depois** — o conteúdo mais persuasivo que existe em fitness
3. **Gera conteúdo para Instagram/TikTok** — "Veja minha transformação em 90 dias"
4. **Viral loop** — quem publica inspira outros a começar
5. **Retenção forçada** — 90 dias usando o app = hábito formado
